// 01 · Structured Search

What does "structured for AI" actually mean?

Parallel strips all the noise and returns exactly what the AI needs: a clean JSON object with the relevant excerpt, source URL, relevance score, and token count. Nothing more.

// 02 · Basis Confidence

How does Parallel know when to trust its own answers?

Most AI either answers confidently or hallucinates confidently. You can't tell which is which. Parallel's Basis framework solves this — every single output field comes with a citation, a reasoning explanation, and a calibrated confidence score.

HIGH confidence = found on an authoritative source = ~97% accurate = skip human review.
LOW confidence = indirect or ambiguous source = ~64% accurate = a human should check this one.

The result: instead of reviewing every field manually, you only review the LOW ones. At scale across thousands of records, this saves enormous amounts of time.

// 03 · Multi-hop Research

Can it handle research that requires multiple steps?

Some research tasks aren't one search. They require finding the right source, navigating to it, reading the result, identifying what's still missing, searching again, and synthesizing everything. Parallel's Task API does this autonomously — navigating any web infrastructure it encounters without any custom training.

Here's a real workflow: Opendoor uses Parallel to research HOA status for every property they buy or sell. Before Parallel, a human researcher spent 10 minutes per property navigating state government portals. Now it takes 2 minutes of verification.

// What is Parallel?
Parallel is web search infrastructure for AI agents. Their APIs are used by companies like Harvey (legal AI), Opendoor (real estate), Modal (cloud compute), and Kepler (financial research) to power accuracy-critical workflows at scale.
Their products include the Search API (fast structured search), the Task API (deep multi-step research with Basis verification), the Monitor API (track changes on any website), and three others — all built specifically for machines, not humans.
